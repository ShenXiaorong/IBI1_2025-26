Reflection for Git and GitHub Practical
This practical helped me master core Git and GitHub skills step by step. I learned the basic workflow of local Git version control: using git init to create a repository, git add to stage files, and git commit -m to save changes with clear messages. I also practiced viewing commit records via git log --oneline, reverting to previous versions with git revert HEAD, and creating and switching branches using git branch and git checkout. For branch collaboration, I finished merging a friend’s branch into the master branch with git merge. On GitHub, I successfully signed up for an account, forked a repository, cloned it to my local computer with git clone, and pushed local edits back to GitHub using git push origin master. Additionally, I built a simple personal website with GitHub Pages, configured a Jekyll theme, and understood how to link the local index.md file to the online webpage.
Overall, the practical was not that smooth. I faced challenges all the time and need to do much extract work beyond following the guidance. Despite these challenges, I admit this practical is quite informative and useful. I now understand that version control is essential for project management, as it tracks every change and enables easy collaboration. This practical laid a solid foundation for my future coding and project work with Git and GitHub.
